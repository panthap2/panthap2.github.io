Issue Description

searchBar: false renders a search bar on iOS 11+.

Steps to Reproduce / Code Snippets / Screenshots

Create a component with the following definition.

{
  component: {
  id: "a.component",
  name: "a.component",
  options: {
    topBar: {
      searchBar: false,
      title: {
        text: "Search Bar is Visible,
      },
    },
  },
}

Without searchBar: false there is no search bar, with searchBar: any, a wild search bar appears.

This is due to

react-native-navigation/lib/ios/RNNViewControllerPresenter.m

Line 31 in b0c8113

if (options.topBar.searchBar.hasValue) {
just checking if there is a value instead of making sure the value is truthy.

I wanted to do a PR, but it's also present here

react-native-navigation/lib/ios/RNNViewControllerPresenter.m

Line 67 in b0c8113

if (newOptions.topBar.searchBar.hasValue) {
and I wasn't sure how to handle removing the search bar.

Environment

  • React Native Navigation version: 2.1.3-snapshot.50
  • React Native version: 0.56
  • Platform(s) (iOS, Android, or both?): iOS
  • Device info (Simulator/Device? OS version? Debug/Release?): iOS 11+
