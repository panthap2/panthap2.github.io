katzyn commented Jul 25, 2020 •
edited

It looks like you have incorrect referential constraint that references set of columns that doesn't have an unique or primary key constraint. Actually such constraint must exists, this is clearly required by syntax rules of referential constraint in the SQL Standard, but when it isn't found, H2, instead of throwing an exception, tries to create such constraint by itself. Usually it works, but when tables are located in different schemas, this synthetic constraint is created in the wrong schema.

CREATE SCHEMA S1;
> ok

CREATE TABLE S1.T1(ID INT);
> ok

CREATE SCHEMA S2;
> ok

CREATE TABLE S2.T2(ID INT, FK INT REFERENCES S1.T1(ID));
> ok

@reconnect

Of course, this fallback code needs to be fixed, but you really should define an unique or primary key constraint for each set of referenced columns by yourself.
