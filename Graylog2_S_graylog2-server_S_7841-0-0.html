We noticed that event definitions do not run as configured (eg. every 1 minutes) after changing the "Search within the last" setting.

Current Behavior

When we create an event definition and set the following fields under "Filter & Aggregation":

  • Search within the last: 1 minutes
  • Execute search every: 1 minutes

... the following mongo document will be created in "scheduler_triggers":

{
	"_id": "5e8c6a0688ea5f254b775049",
	"job_definition_id": "5e8c6a0688ea5f254b775048",
	"start_time": "2020-04-07T11:54:46.852Z",
	"end_time": null,
	**"next_time": "2020-04-07T11:55:46.851Z",**
	"created_at": "2020-04-07T11:54:46.852Z",
	"updated_at": "2020-04-07T11:54:46.852Z",
	"triggered_at": "2020-04-07T11:54:52.546Z",
	"status": "runnable",
	"lock": {
		"clock": "0",
		"progress": 0,
		"last_lock_time": "2020-04-07T11:54:52.546Z",
		"owner": null
	},
	"schedule": {
		"type": "interval",
		"interval": "60000",
		"unit": "MILLISECONDS"
	},
	"data": {
		"type": "event-processor-execution-v1",
		"timerange_from": "2020-04-07T11:54:46.852Z",
		"timerange_to": "2020-04-07T11:55:46.851Z"
	}
}

So far, so good. Everything works as expected and the scheduler runs every minute. After we change the event definition to:

  • Search within the last: 5 hours
  • Execute search every: 1 minutes

... the mongo document will be changed to:

{
	"_id": "5e8c6a0688ea5f254b775049",
	"job_definition_id": "5e8c6a0688ea5f254b775048",
	"start_time": "2020-04-07T11:54:46.852Z",
	"end_time": null,
	**"next_time": "2020-04-07T16:55:46.852Z",**
	"created_at": "2020-04-07T11:54:46.852Z",
	"updated_at": "2020-04-07T11:56:12.494Z",
	"triggered_at": "2020-04-07T11:55:52.296Z",
	"status": "runnable",
	"lock": {
		"clock": "0",
		"progress": 0,
		"last_lock_time": "2020-04-07T11:55:52.296Z",
		"owner": null
	},
	"schedule": {
		"type": "interval",
		"interval": "60000",
		"unit": "MILLISECONDS"
	},
	"data": {
		"type": "event-processor-execution-v1",
		"timerange_from": "2020-04-07T11:55:46.852Z",
		"timerange_to": "2020-04-07T16:55:46.852Z"
	}
}

Notice the "next_time" field. After changing the event definition, the scheduler will only run this job 5 hours after the change, even though it should still be running every minute. After 5 hours the job will run every 1 minute again.

Expected Behavior

Changing "Search within the last" should not affect the time on which the job will run next.

Steps to Reproduce (for bugs)

  1. Create a new event definition
  2. Under "Filter & Aggregation" set "Search within the last" to "1 minutes" and "Execute search every" to "1 minutes"
  3. Save the definition
  4. Open the event definition and change the value for "Search within the last" to "5 hours"

Your Environment

  • Graylog Version: 3.1.4
  • Elasticsearch Version: 6.8.7
  • MongoDB Version: 4.0.16
  • Operating System: CentOS 7
  • Browser version: Firefox 75
