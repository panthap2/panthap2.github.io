Comment #1

DavidDTA commented Feb 10, 2020

Describe the bug
After composition, this animation does not render properly.
After composition and setting the progress to 0, it does not render properly, due to the check in LottieValueAnimator.setFrame which makes it a no-op if the frame doesn't change.
After composition and setting the progress to some nonzero value and then back to 0, it does render properly.

It appears that there is an issue in composition which causes the animation's state not to be exactly the same as it would be after setting progress to 0.

I am working around the issue by setting the progress to 1 and then back to 0 after composition.

Steps To Reproduce

  • Load this animation into a LottieAnimationView.
  • Notice that you see an egg without a crack. You should see an egg with a crack.
  • Notice that the progress is 0.
  • Set the progress to 0.
  • Notice that the rendering does not change.
  • Set the progress to .5 and then back to 0.
  • Notice that you see an egg with a crack as expected.

This can be seen in the issue-repro app with the following patch:
