Comment #5

orionlee commented Sep 28, 2019 •
edited

@johannes You can follow the build instruction . Basically you need to set up your own key for the release build.

But building a release build probably won't help you, as the one you build can't exactly replace the official build, unless you are okay to uninstall the official one.

You could follow get the logcat details of the existing installation by finding out the PID of the running AntennaPod and use it to filter out.

For example:

  1. attach your device to your computer
  2. open android studio's Logcat window
  3. ask it to filter something that AntennaPod uses, e.g., PlaybackService
  4. Have AntennaPod play something, you would see lines such as
2019-09-28 16:28:27.963 5419-8698/? D/PlaybackService: Notification set up
2019-09-28 16:28:32.936 5419-5419/? D/PlaybackService: Saving current position to 1033645
  1. You could then tell the PID of the running AntennaPod. It is 5419 in the above example.
  2. Now you can filter out with the PID (5419 above) to get all AntennaPod logs.

draeath commented Sep 29, 2019

Note you can obtain the linux UID of the package with this: adb shell dumpsys package de.danoeh.antennapod | grep userId=

You can then do adb shell ps -u UID where UID is the number the previous command returned. This will return all running process IDs belonging to this UID.

Saves you having to do anything with the application and logcat to find it's PID...
