chanseokoh commented Jan 10, 2020 •
edited

Jib (JVM) failed to verify the server over HTTPS:

[WARNING] Cannot verify server at https://192.168.1.2:2376/v2/. Attempting again with no TLS verification.

So it attempts the port again with no TLS verification. But it also failed even when not verifying the server:

[WARNING] Failed to connect to https://192.168.1.2:2376/v2/ over HTTPS. Attempting again with HTTP.

So Jib attempted speaking plain-HTTP on the port 2376. I suspect the port 2376 doesn't speak HTTP, so that's why you get ClientProtocolException and the message "(t)he server failed to respond with a valid HTTP response."

What does curl -v -k https://192.168.1.2:2376/v2/ say? Maybe write a simple Java program to connect to https://192.168.1.2:2376/v2/ and see if it works. I wonder if the server uses some kind of SSL-related algorithm that the JVM cannot handle. Another possibility is that the port 2376 doesn't speak TLS/SSL at all.

BTW, note that in general, if you installed the self-signed certificate to the JVM truststore correctly, you don't even need to set allowInsecureRegistries.
