JonLally commented Jul 10, 2020 •

Can confirm this issue is in version 6.5.0 and believe it is related to the fix for #2853 . Was able to reproduce the issue with a small sql for postgres with placeholders

CREATE TABLE people (
    id SERIAL primary key,
    first_name CHARACTER VARYING(50),
    surname CHARACTER VARYING(50),
    country CHARACTER VARYING(50)
);

INSERT INTO people (first_name, surname, country) VALUES ('${first_person_first_name}', '${first_person_surname}', '${first_person_country}');

When I provide the placeholder values 'John', 'Doe' and 'Ireland' respectively I get the exception reported above. I think this is because those placeholder values make the script 245 characters long, 55 less than its original 300. The PeekingReader.refillPeekBuffer() method keeps adding to the peekBuffer after 245 characters is reached. When reset() is called the BufferedReader throws the exception because markedChar is now -2. If I remove the check !supportsPeekingMultipleLines inside PeekingReader.refillPeekBuffer() the script gets migrated successfully.
