Comment #19

pshipton commented Apr 23, 2020

We discovered more information.

  • The problem is not related to the shared classes cache. The reason it occurs when shared classes is enabled is due to the shared cache storing debug information from the class. If the shared cache is enabled but created with -Xnolinenumbers (), the problem should not occur because the debug information from the class will not be parsed and verified.
  • Likely the problem can be created with Hotspot when debug is enabled (-Xrunjdwp:transport=dt_socket,address=8888,server=y,suspend=n), meaning it's a problem with the class file and not with OpenJ9.
