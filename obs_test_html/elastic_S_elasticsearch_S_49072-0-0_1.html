Comment #14

daniele-pini commented Feb 4, 2020

Hi all, I've been investigating the issue with @akhileshbhatia, looking for temporary workarounds.

The execution_hint just doesn't apply here, since the reported query uses a sampler aggregation, while the execution_hint seems to apply only for diversified_sampler aggregations.

Apart from that, I have noticed that the problem seems to disappear (in all cases I tested) if I happen to add a dummy 1-bucket terms aggregation right before the sampler one, like this:

"aggs": {
  "dummy_terms_wrapper": {
    "terms": {
      "field": "__not_present__",
      "missing": ""
    },
    "aggs": {
      "sampler": {
        "sampler" ...
      }
    }
  }
}

there's a small performance hit in adding this layer, but it seems to solve the problem in our cases.

No real idea if this actually works in ALL cases, or why though - so take this 'workaround' with a grain of salt.
