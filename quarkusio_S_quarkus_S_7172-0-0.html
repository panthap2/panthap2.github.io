Describe the bug
I am encountering an issue when sending a large set of data to the Quarkus java server. It appears that I am hitting the max allowed frame size for vertx, and I cannot find any way to configure it for Quarkus. From the digging I have done thus far, I think the culprit is io.vertx.core.http.HttpServerOptionsmaxWebsocketFrameSize, which seems to be using the default value.

Expected behavior
Ability to configure Quarkus to support a larger max frame size, and transmit data without encountering any errors.

Actual behavior
Unable to transmit data larger than 65536 bytes. Error occurs during transmission, and the server never receives the data.

To Reproduce
Steps to reproduce the behavior:

  1. Publish a string message over websocket larger than 65536 bytes.

Configuration
I have tried changing the http limits max-body-size and max-header-size, no luck.

Environment (please complete the following information):

  • Output of uname -a or ver: Linux my-server 3.10.0-957.12.1.el7.x86_64 1 SMP Wed Mar 20 11:34:37 UTC 2019 x86_64 Linux
  • Output of java -version: OpenJDK Runtime Environment AdoptOpenJDK (build 11.0.5+10)
  • Quarkus version or git rev: 1.1.1.Final

Additional context
Server is running on a docker container deployed through kubernetes. The websocket endpoint is created using javax.websocket annotations.
